# 前端错误排查与布局检查

为 Vue 2 + Element UI 项目提供两种核心能力：

1. **运行时错误排查** — 根据报错信息/截图定位问题根因，给出修复方案
2. **布局还原检查** — 对比页面实现与设计稿，找出布局偏差并给出修改建议

## 能力一：运行时错误排查

### 接收报错信息

用户可能通过以下方式提供报错：
- 直接粘贴控制台错误信息文本
- 发送报错截图
- 口头描述现象（如"页面白屏了"、"点击没反应"、"接口报 500"）

不管哪种方式，核心思路是一样的：**从报错信息出发，定位到源码位置，分析根因，给出修复方案。**

### 排查流程

#### Step 1: 识别错误类型

先判断报错属于哪一类：

| 错误类型 | 典型表现 |
|---------|---------|
| Vue 渲染错误 | `Error in render: "Cannot read property 'xxx' of undefined"` |
| 事件处理错误 | `Error in v-on handler: "xxx is not a function"` |
| 网络请求错误 | 接口返回 4xx/5xx，或请求本身失败（超时、CORS） |
| 路由错误 | `NavigationDuplicated`，路由跳转白屏 |
| Element UI 组件错误 | `el-table` 渲染异常，`el-form` 校验不生效，`el-dialog` 打不开 |
| 样式/布局错误 | 组件渲染了但位置、大小、样式不对 |
| 生命周期错误 | `Error in mounted/created hook`，`this` 指向问题 |
| 状态管理错误 | Vuex state undefined，mutation/action 报错 |

#### Step 2: 定位源码

根据错误堆栈中的文件名和行号，去项目中找到对应的代码文件。重点看：
- 报错堆栈中指向的 `.vue` 文件
- 如果是接口错误，找到对应的 API 文件和调用处
- 如果是组件错误，检查组件的 props、data、computed、methods

**注意**：Vue 2 的错误堆栈经常指向编译后的代码，需要根据组件名和方法名反推到源码位置。

#### Step 3: 分析根因

常见的 Vue 2 + Element UI 项目错误模式：

**数据未初始化**
```js
// 报错：Cannot read property 'name' of undefined
// 原因：后端返回的 user 可能为 null
// 修复：提供默认值或可选链
this.formData = res.data.user || {}
// 模板中：(user || {}).name 或 v-if="user"
```

**异步时序问题**
```js
// 报错：Cannot read property 'xxx' of undefined
// 原因：在 data 中引用了异步数据，但模板先于数据渲染
// 修复：初始化默认值 + v-if 守卫
data() {
  return {
    detail: {}  // 不要写成 null
  }
}
// 模板中：<div v-if="detail.id">{{ detail.name }}</div>
```

**this 指向丢失**
```js
// 报错：this.xxx is not a function
// 原因：回调函数中 this 不是 Vue 实例
// 修复：使用箭头函数或提前绑定
// 错误写法
setTimeout(function() { this.getList() }, 1000)
// 正确写法
setTimeout(() => { this.getList() }, 1000)
```

**Element UI 表格渲染异常**
```js
// 报错：el-table 渲染空白或报错
// 常见原因：
// 1. :data 绑定的不是数组
// 2. el-table-column 的 prop 对应字段不存在
// 3. slot-scope 中的 scope.row 为 undefined
// 修复：确保 data 是数组，加 v-if 判断数据加载完成
```

**Element UI 表单校验不生效**
```js
// 常见原因：
// 1. el-form-item 缺少 prop 属性
// 2. prop 的值和 v-model 绑定的字段名不一致
// 3. rules 中没有对应字段的规则
// 4. 自定义 validator 中忘记调用 callback()
```

**接口请求错误**
```js
// 4xx 错误：检查请求参数、权限、接口路径
// 5xx 错误：后端问题，但可以先确认请求参数格式是否正确
// 网络错误：检查代理配置、CORS、 baseURL
// 重点检查：src/utils/request.js 中的拦截器逻辑
```

#### Step 4: 给出修复方案

修复方案应该包含：
1. **问题原因**：一句话说明为什么出错
2. **修复代码**：给出具体的代码改动，而不是笼统的方向
3. **验证方式**：改完之后怎么确认修好了

如果问题有多种修复方式，优先选择最简单、侵入性最小的方案。

## 能力二：布局还原检查

### 接收输入

用户可能提供：
- 页面截图（实际效果）
- 设计稿截图
- 设计平台链接（Figma、蓝湖等）

如果是设计平台链接，需要用户协助获取截图（目前无法直接访问设计平台）。

### 检查流程

#### Step 1: 对比整体布局

先从宏观角度对比：
- **页面结构**：整体布局是否一致（上下结构、左右结构、栅格布局）
- **内容区域**：各模块的位置、顺序是否正确
- **间距比例**：模块之间的间距是否合理

#### Step 2: 检查具体细节

逐一对比以下维度：

| 检查维度 | 具体内容 |
|---------|---------|
| 间距 | margin、padding 是否和设计稿一致 |
| 字体 | 字号、字重、行高、颜色 |
| 颜色 | 背景色、文字色、边框色、主题色 |
| 尺寸 | 宽度、高度是否精确 |
| 圆角 | border-radius 是否一致 |
| 阴影 | box-shadow 是否一致 |
| 边框 | border 宽度、样式、颜色 |
| 图标 | 图标类型、大小、颜色 |
| 对齐方式 | 水平/垂直对齐、flex 布局方向 |

#### Step 3: 定位样式代码

根据差异点，找到对应的源码文件和样式代码：
- 找到对应的 `.vue` 文件
- 找到具体的 `<style>` 部分或独立的样式文件
- 给出需要修改的 CSS 属性和具体值

#### Step 4: 输出差异报告

按优先级列出差异：

```
## 布局还原差异

### 严重（结构/位置错误）
1. [问题描述] → 修复：[具体CSS改动]

### 中等（间距/尺寸偏差）
2. [问题描述] → 修复：[具体CSS改动]

### 轻微（颜色/字体细节）
3. [问题描述] → 修复：[具体CSS改动]
```

## 通用注意事项

### Vue 2 + Element UI 专项检查清单

排查任何问题时，留意这些常见坑：

- **Vue 2 限制**：不支持可选链 `?.`（除非 Babel 配置了），用 `&&` 或默认值代替
- **响应式丢失**：Vue 2 无法检测属性的新增/删除，需要 `this.$set()`
- **v-for 缺少 key**：列表渲染必须加 `:key`，且不要用 index 做 key
- **el-select 远程搜索**：需要配合 `filterable` + `remote` + `remote-method`
- **el-dialog 嵌套**：内层 dialog 需要加 `append-to-body`
- **el-table 动态列**：列变化时可能需要加 `:key` 强制重新渲染
- **$nextTick**：DOM 更新后才能操作，需要在 `this.$nextTick()` 回调中获取 DOM 元素
- **深拷贝问题**：直接赋值对象是引用，修改会影响原数据，用 `JSON.parse(JSON.stringify())` 或展开运算符

### 输出规范

- 修复代码必须符合 Vue 2 Options API 语法，不用 `<script setup>` 或 Composition API
- 样式修改使用项目的 CSS 预处理器（如果没有就用纯 CSS）
- 不要使用 `!important`，通过选择器权重解决样式覆盖
- 不要直接覆写 Element UI 组件原生样式，除非用户明确要求
- 代码使用两空格缩进
- 异步操作使用 `async/await`，不用回调写法
